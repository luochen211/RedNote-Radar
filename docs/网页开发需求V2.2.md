## 1. 整体风格与技术栈
### 1.1 设计风格
- **主色调**：**暗黑/灰黑背景** + **天蓝/霓虹蓝点缀** (Dark/Grey + Neon Blue)。
- **视觉感受**：极具现代感的科技风 (Tech/Futuristic)，配合玻璃拟态 (Glassmorphism) 效果。
- **UI 组件库**：推荐使用 **Shadcn/ui** (基于 Radix UI + Tailwind CSS)，自带高质量暗黑模式组件。
- **图表库**：**Recharts** 或 **Visx**，用于实现雷达图与自定义交互浮窗。

### 1.2 多语言支持 (i18n)
- **默认语言**：英文 (English)。
- **切换支持**：中文 (Chinese)。
- **内容策略**：专业术语无需深度本地化，保持英文准确性，界面标签支持双语切换。

### 1.3 项目命名
- **中文名**：酒店短视频智算台
- **英文名**：Hotel Video InsightHub

## 3. 页面详细需求

### 3.1 首页 (Home)
- **登录区 (Login Section)**：
  - **组件**：普通的账号/密码登录框。
  - **技术实现**：集成 **Supabase Auth** 或 **NextAuth.js**。
  - **鉴权**：普通用户登录后跳转 Upload 页；**管理员**登录后导航栏出现 "Admin" 选项。
  - **验证码**：简单的图形/数学验证码，防自动化爆破。
- **核心展示区 (Hero & Intro)**：
  - **简介**：展示核心算法 (VGG/VGGish/BERT)、五大模块 (Feature Extraction, Interaction, Fusion, Attention, Prediction)。
  - **架构图**：高清展示模型框架图。
- **友情链接 (Footer)**：
  - 黑色底栏，包含：李恒云教授主页、慕杰主页、Hotel Icon 官网/小红书、PolyU SHTM 官网。

### 3.2 数据上传页 (Data Upload)
- **布局**：居中表单，Glassmorphism 风格卡片。
- **表单字段 (8类)**：
  1. **视频 (Video)**: 支持 MP4，限制大小 (如 100MB)，显示上传进度。
  2. **首图 (Cover Image)**: JPG/PNG，预览缩略图。
  3. **标题 (Title)**: 单行文本。
  4. **文本内容 (Text)**: 多行文本区域。
  5. **标签 (Tags)**: 动态 Tag 输入器 (Enter 添加，点击 x 删除)。
  6. **粉丝数 (Followers)**: 数字输入。
  7. **关注数 (Subscribers)**: 数字输入。
  8. **获赞数 (Total Likes)**: 数字输入。
- **交互**：点击 "Submit" 后，先进行前端基础校验，通过后锁定按钮并跳转。

### 3.3 预测结果页 (Prediction)
- **加载状态 (Loading State)**：
  - **技术实现**：使用 **Server-Sent Events (SSE)** 实现实时进度推送。
  - **UI**：进度条或步骤指示器 (Stepper)，显示 "提取视频帧..." -> "分析音频..." -> "生成预测..."。缓解用户 1 分钟的等待焦虑。
- **结果展示**：
  - **两大维度**：
    1.  **Local Scope**: 基于 Hotel Icon历史数据，预测相对高互动概率。
    2.  **Global Scope**: 基于全行业数据，预测成为爆款的概率。
  - **形式**：仪表盘 (Gauge Chart) 或大号数字，配文字解读。

### 3.4 分析结果页 (Analysis)
- **Tab 导航**：
  1.  **Content Quality Scores**
  2.  **Content Sentiment Scores**
  3.  **Content Consistency Scores**
  4.  **Oriental Aesthetics Score**
- **交互要求**：
  - 所有指标名称悬停 (Hover) 时，显示**聊天气泡风格浮窗**，内容包括：中英文定义 + 取值范围。
  - 数值范围统一：前端展示 **0-100** (若后端返回0-1需自动 x100)。
- **详细指标可视化**：
  - **Quality**: 6项。数值型用进度条/环形图；Bool型 (人声/人脸) 用 "亮灯/灭灯" 拟物图标。
  - **Sentiment**: 5项。柱状图或雷达图。
  - **Consistency**: 6项。匹配度百分比。
  - **Oriental Aesthetics**: **双三角雷达图**。
    - **Radar 1 (Aesthetics)**: Richness, Harmony, Adaption.
    - **Radar 2 (Cultural)**: Modern, Oriental, Western.
    - **样式**：暗色背景，雷达区域半透明填充，不同颜色描边区分。

### 3.5 管理员后台 (Admin Dashboard)
- **权限**：仅 `role = 'admin'` 用户可访问 (路由中间件保护)。
- **功能模块**：
  - **用户管理**：列表展示，支持新增、删除用户。
  - **数据审计**：查看所有用户上传的历史记录、导出数据。
  - **日志系统**：简单的操作日志 (Who, When, What)。
- **实现建议**：复用 Shadcn 的 Table 组件 (Data Table)。

## 5. 通用与运营要求
- **数据一致性**：提交后不可修改 (Immutable)，只能重新上传。所有历史记录在 Upload 页可查。
- **任务并发**：预期并发 < 1000 (极低)。SSE 保持连接即可。
- **数据清理**：后端定时任务 (Cron Job) 每 3 个月清理旧的视频文件，但**保留**元数据 (Metadata) 和分析结果用于统计。

## 6. 部署与存储方案 (技术升级)

为解决**数据持久化** (V2 原版 SQLite 在 Serverless 环境丢失数据) 和**成本控制**问题，采用以下架构：

### 6.1 架构图
- **Frontend**: Next.js (App Router) --> 部署于 **Vercel** (Global CDN)。
- **Backend API**: Python (FastAPI/Flask) + PyTorch --> 部署于 **Modal**
- **Database**: **PostgreSQL** --> 部署于 **Supabase** (免费层 500MB，支持自动备份)。
- **Auth**: **Supabase Auth** (集成 Email/Password 登录，自带 User管理)。
- **Storage**: **Supabase Storage** (用于存视频/图片)。

### 6.2 选型优势
1.  **Supabase vs SQLite**: 
    - 解决了 Vercel 无法保存 SQLite 文件的问题。
    - 免费层足够支持本项目（500MB 数据库，1GB 文件存储，50,000 MAU）。
    - 提供了现成的 Admin 面板，减少 "管理员页面" 30% 的开发工作量。
2.  **Next.js + Vercel**:
    - 极速的全球访问 (Edge Network)。
    - 对 SEO 友好。
3.  **SSE (Server-Sent Events)**:
    - 替代轮询，更优雅地处理 "1分钟预测" 的等待体验。

### 6.3 域名与预算
- **域名**: 购买 `smart-tourism-hospitality.com` (约 $10-20/年)。
- **服务器成本**:
  - Frontend: $0 (Vercel Hobby)。
  - Database: $0 (Supabase Free)。
  - GPU Backend: 按量付费 (Modal/AWS)，预估 < $5/月 (基于日均 20 次调用)。
  - **总计**: 约 $20/年 (主要是域名费)
